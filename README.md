# java-kanban
Выполнено с использованием как id hashCode, также при замене(апгрейте), сохраняются старые айди даже при вносе полностью нового объекта, тк в дальнейшем, возможно нам будут важны позиции айди и изначальный порядок задач
Также не везде использовалась перессылка айди для апгрейта объекта, так как при использовании хешМепа айди неудобен для использования, вместо него передается объект, а оттуда берется айди, надеюсь в этот раз будет более удачно:-)
